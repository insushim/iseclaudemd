### 2. 스토리 구조

#### 3막 구조
```
1막: 설정 (25%)
├── 일상 세계
├── 사건의 발단
└── 결심/모험의 시작

2막: 대립 (50%)
├── 시련과 성장
├── 중간 지점 반전
├── 최대 위기
└── 암흑의 순간

3막: 해결 (25%)
├── 클라이맥스
├── 대단원
└── 새로운 일상
```

#### 기승전결 (한국식)
```
기(起): 이야기의 시작, 배경과 인물 소개
승(承): 사건 전개, 갈등 고조
전(轉): 반전, 위기의 정점
결(結): 해결, 결말
```

### 3. 캐릭터 개발

#### 캐릭터 시트
```markdown
## 기본 정보
- 이름:
- 나이:
- 외모:
- 직업/역할:

## 성격
- MBTI:
- 강점:
- 약점:
- 습관/버릇:

## 배경
- 가족관계:
- 과거사:
- 트라우마:

## 동기
- 원하는 것(Want):
- 필요한 것(Need):
- 두려움:

## 성장 아크
- 시작점:
- 변화 계기:
- 도착점:
```

#### 캐릭터 유형
```yaml
protagonist_types:
  - 영웅: 정의롭고 용감
  - 안티히어로: 결함 있는 주인공
  - 언더독: 약자에서 강자로
  - 트라지 히어로: 비극적 결말

supporting_types:
  - 멘토: 지혜를 전달
  - 조력자: 주인공을 돕는 친구
  - 연인: 로맨스 상대
  - 라이벌: 경쟁자
  - 트릭스터: 예측불허 인물

antagonist_types:
  - 빌런: 명확한 악역
  - 그레이 캐릭터: 모호한 선악
  - 시스템: 사회/제도적 악
  - 내면의 적: 자기 자신
```

### 5. 문장과 문체

#### 시점별 특징
```yaml
1인칭:
  장점: 몰입감, 감정 전달
  단점: 정보 제한
  용도: 심리 묘사, 미스터리

3인칭 제한:
  장점: 객관성 + 내면 묘사
  단점: 시점 인물 제한
  용도: 가장 보편적

3인칭 전지:
  장점: 모든 정보 전달 가능
  단점: 몰입감 저하
  용도: 대서사, 군상극

다중 시점:
  장점: 입체적 스토리
  단점: 혼란 가능
  용도: 복잡한 플롯
```

#### 문체 가이드
```markdown
## 묘사의 원칙
- Show, Don't Tell
  ❌ 그녀는 화가 났다.
  ✅ 그녀의 주먹이 하얗게 질렸다.

## 대화의 원칙
- 각 캐릭터만의 말투
- 서브텍스트 활용
- 적절한 지문 배치

## 리듬의 원칙
- 긴장감: 짧은 문장
- 서정적: 긴 문장
- 변화 주기
```

## 실전 예시

### 판타지 오프닝 예시
```
[평범한 일상]
대학 도서관 한구석. 이건우는 취업 스터디 노트를 뒤적이다 한숨을 내쉬었다.

[사건 발생]
'띠링.'
눈앞에 떠오른 반투명한 창. 처음에는 피로 때문인 줄 알았다.

[시스템 창]
『축하합니다. 당신은 '회귀자'로 선택되었습니다.』
『10년 후의 기억을 보유한 채 과거로 돌아갑니다.』

[반응]
"...뭐?"
이건우는 멍하니 그 글자를 읽었다. 그리고 다음 순간—
세상이 반전되었다.
```
